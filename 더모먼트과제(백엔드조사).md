# 백엔드란?
<b>백엔드는 웹사이트나 앱에서 눈에 보이지 않는 뒷단(서버,데이터베이스,API)을 말한다.</b>
<br><br>
# 백엔드를 구성하는 주요 요소
### 1.서버
서버는 데이터 베이스,응용 프로그램 등 과 상호작용하며 클라이언트의 요청을 처리하고 응답하는 컴퓨터이다.
### 2.애플리케이션
비즈니스 로직을 실행하는 프로그램(데이터 처리,사용자 인증, 비즈니스 로직을 담당하는 핵심 시스템)
```
비즈니스 로직 : 비즈니스 로직은 쉽게 말해 서비스가 돌아가는 규칙(결정을 내리는 공식)
```
### 3.데이터베이스
데이터베이스는 구조화된 정보 또는 데이터의 조직화된 모음<BR>
쉽게 말하면 데이터 베이스는 `데이터 집합소(데이터를 모아놓은 커다란 창고)`

##### DBMS
```
DBMS(데이터베이스 관리 시스탬)는 데이터 베이스를 관리하는 소프트웨어

DBMS가 하는일
->CRUD(CREATE생성,READ읽기,UPDATE수정,DELETE삭제),보안,동시접속 제어

DBMS를 사용하는 이유
-> 수많은 사람이 동시에 접속해도 데이터가 꼬이거나 사라지지 않게, 그리고 엄청나게 빠르게 관리하려고
```
##### SQL
```
SQL은 DBMS에서 데이터를 정의,조작,제어하기 위해서 사용하는 표준프로그래밍 언어
-> 쉽게 말해 데이터 베이스의 소통방식이다.

SQL 명령어
- DDL (데이터 정의어)
    CREATE 만들기,ALTER 고치기,DROP 없애기,TRUNCATE 비우기

- DCL (데이터 제어어)
    GRANT 허락하기,REVOKE 뺏기

- DML (데이터 조작어)
    INSERT 넣기,DELETE 빼기,SELECT 보기,UPDATE 바꾸기
```
### 4.API
API는 다른 소프트웨어 시스템과 통신하기 위해 따라야하는 규칙

#### REST API
API는 프론트엔드(클라이언트)의 요청을 받아서 데이터 베이스와 상호작용하는 인터페이스


# 프론트엔드 VS 백엔드 차이점
| 구분 | 프론트엔드 | 백엔드|
|------|-----------|-------|
|역할|UI 구현,사용자 입력처리|서버로직처리,데이터 저장,인증 등||
|사용언어|HTML,CSS,JavaScript...|Java,Spring,Python...|
|동작위치|클라이언트|서버|

# 백엔드 개발자가 하는일
- API 서버 개발
<BR>
프론트의 요청을 받아 처리 할수있게 REST API,GraphQL을 설계,구현

- DB 설계 및 관리
<BR>
사용자 정보,게시글 등 다양한 데이터를 저장하고 조회할수있게 데이터 베이스 설계
- 인증,보안처리
<BR>
로그인,권한 분리 등 사용자의 정보 보호를 위한 보안 작업 수행
- 비즈니스 로직 처리
<BR>
결제,예약,포인트 계산 등 실제 서비스 기능 중심 로직 처리


## 데이터 흐름
<B>사용자 요청(인터페이스의 요청) -> 백엔드 서버 -> DB조회,처리 -> 클라이언트에 응답</B>

예시
```
배민 주문 예시

1. 사용자 요청 
배민앱에 들어가서 치킨 시킴

2.백엔드 서버 
치킨집이 영업 하고있는가?
할인 쿠폰을 사용했는가?
서비스 운영 규칙 적용(비즈니스 로직)

3.DB조회,처리 
쿠폰 사용(UPDATE),주문내역 추가(INSERT)

4.클라이언트에 응답
서버에서 주문성공이라는 데이터를 JSON형식으로 보냄
프론트엔드가 화면에 주문 완료라는 글을 띄움
```
JSON : 컴퓨터끼리 데이터를 주고 받을떄 사용하는 가장 대표적인 메모양식
