# HTTP
HTTP(Hypertext Transfer Protocol)는 웹 브라우저(클라이언트)와 웹 서버가 HTML, 이미지 등 데이터를 주고받기 위한 통신 규약이다.

# HTTP가 웹에서 하는일
HTTP는 브라우저와 서버간에 데이터를 교환하는데 사용된다.
```
인스타그램으로 예시를 들면
친구가 올리 게시물에 내가 하트를 누르면 브라우저(클라이언트)가 "친구의 게시물에 하트 하나 추가해줘" 라고 HTTP 요청을 서버에 보낸다.
그리고 서버로부터 응답이 오면 화면에 하트가 눌러짐
```
# Stateless 구조
Stateless(무상태성) 구조는 말 그대로 서버가 클라이언트 이전 상태를 기억하지 않는다 라는 뜻이다.

무상태(Stateless) 프로토콜이 필요한 이유
-> 서버의 부담 감소 : 서버가 수억명의 상태를 다 기억하면 서버 과부하로 터질수도있다

그러나 매번 기억 못하는건 아니다 쿠키와 토큰 등을 이용한다



# 클라이언트-서버 구조
클라이언트 서버 구조는 Request Response 구조로 클라이언트는 서버에 요청을 보내,응답을 대기한다. 서버가 요청에 대한 결과를 만들어 대답한다.

클라이언트와 서버와 개념적으로 분리를 한다.
비즈니스 로직이랑 데이터는 서버에 다 밀어넣는다.
클라이언트는 UI그리고 사용성에만 집중을 한다.

이렇게하면 클라이언트와 서버가 각각 독립적으로 진화할 수가 있다.

# HTTP Method란?
HTTP 메서드는 클라이언트(브라우저)가 서버에 보낼 때 ` 내가 지금 어떤 의도로 이 요청을 하는 거야 `라고 알려주는 행동 지침이다.

# 대표적인 HTTP 메서드
| HTTP 메서드 | 의미 | 목적 | 데이터 베이스 작업|
|----|----|----|----|
|GET|조회|서버에서 데이터를 가져오기|Read|
|POST|생성|새로운 리소스를 생성하거나 데이터를 서버로 제출할 때 사용|Create|
|Put|수정|리소스의 전체를 교체(수정)할 때 사용 (존재하지 않으면 생성)|Update|
|DELETE|삭제|리소스를 삭제할 때 사용|Delete|

- PATCH(일부 수정)리소스의 부분적인 내용을 수정할 때 사용
<br><br><br>
# HTTP Status code란?
HTTP Status code는 클라이언트(브라우저 등)가 보낸 요청의 처리 결과를 서버가 숫자로 응답하는 3자리 코드이다

자주 나오는 상태 코드
- 2xx (성공)
  - 200 OK : 요청이 성공
  - 201 Created : 요청이 성공하여 새로운 리소스가 생성됨
- 3xx (리다이렉션)
  - 301 Moved Permanently: 요청한 리소스의 URI가 영구적으로 변경됨
  - 302 Found: 요청한 리소스가 일시적으로 다른 URI에 존재함
- 4xx (클라이언트 에러)
  - 400 Bad Request: 잘못된 문법 등으로 서버가 요청을 이해하지 못함
  - 401 Unauthorized: 인증(로그인 등)이 필요함

- 5xx (서버 에러)
   - 500 Internal Server Error: 서버 내부 오류 (코드 에러 등)
   - 503 Service Unavailable: 서버 점검이나 과부하로 서비스 이용 불가


# HTTP 요청과 응답의 구조
HTTP의 요청과 응답은 헤더(Header)와 본문(Body) 두가지 요소로 나뉜다. 이 두요소는 클라이언트와 서버 간의 데이터를 주고 받을때 서로 다른 역할을 수행한다.

### Header란?
헤더는 클라이언트와 서버가 어떤 데이터를 주고받을지, 그 데이터를 어떻게 처리할지 미리 약속하는 지시문

예시)내가 보내는 데이터가 어떤 형식인지
     내가 누구인지(사용자 정보) 서버에 어덯게 알려줄지
     서버가 응답으로 보내는 데이터를 어떻게 해석할지

#### Header의 역할
Header은 데이터를 어떻게 보내고 받을지 정의한다.(데이터 형식,사용자 정보,인증 정보,캐시 설정 등)

### Body란?
본문(body)은 실제 데이터를 담고 있는 부분
클라이언트와 서버가 실제로 주고받는 내용은 여기에 포함된다

예시
```
fetch('https://www.example.com/login', {
    method: 'POST', // HTTP 메서드를 POST로 설정
    headers: {
        'Host': 'www.example.com', // 요청이 향하는 서버의 도메인 (일반적으로 자동 설정됨)
        'Content-Type': 'application/json' // 서버에 보낼 데이터가 JSON 형식임을 명시
    },
    body: JSON.stringify({
        username: 'user1',
        password: 'pass123'
    }) // 본문에 JSON 형식으로 데이터를 포함
})
```
