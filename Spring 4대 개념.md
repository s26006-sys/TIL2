# Spring 4대 개념

# IoC (제어의 역전)
객체의 객체의 생성, 관리, 흐름을 개발자가 아니라 외부(프레임워크)가 맡는다.

## IoC의 장점
1. 유연성 : 구현체를 바꾸기 쉽다.
<br>
`예시) 데이터 베이스에서 B데이터 베이스로 바꿀때 코드를 최소화 해준다.`

2. 재사용성: 모듈이 독립적이어서 여기저기 가져다 쓰기 좋다.

3. 테스트 용이성: 가짜 객체(Mock)를 주입하기 쉬워져서 단위 테스트가 편해진다.

        프레임 워크 : 프로그래밍에서는 소프트웨어를 효율적으로 개발하기 위한 미리 만들어진 구조와 규칙의 집합(뼈대,틀)
# DI (의존성주입)
IoC를 구현하는 패턴, 필요한 객체(의존성)를 내가 만들지 않고, `외부에서 '주입' 받아서 사용한다.`

# IoC와 DI의 관계
IoC(개념) : `객체의 주도권을 내가 갖지 않고 외부에 넘기겠다`는 철학
<br><br>
DI(도구) : 그 철학을 실현하기 위해 `필요한 객체를 외부에서 넣어주는` 구체적인 기술.

# AOP (관점 지향 프로그램)
AOP는 핵심 로직과 공통 기능을 분리한다.<BR>
로깅(기록), 보안 체크, 트랜잭션(DB 처리)처럼 여러 군데에서 공통으로 쓰이는 코드들을 한곳에 모아서 관리합니다.

# PSA (일관된 서비스 추상화)
특정 기술에 종속되지 않도록, `Spring이 추상화(인터페이스)된 계층을 제공`하여 기술이 바뀌어도 코드가 영향을 받지 않게 한다.

# 마무리
<h2>Spring은 객체지향을 극대화한 프레임워크이다.<br><br>
1. IoC/DI (의존성 주입): 객체들을 '느슨하게' 연결하여 유연하고 테스트하기 쉬운 구조를 만들 수 있다.<br><br>
2. AOP (관점 지향): 공통 부가 기능을 분리하여 개발자가 '핵심 로직'에만 집중하게 한다.<br><br>
3. PSA (서비스 추상화): 특정 기술에 종속되지 않고 '일관된' 개발 방식을 제공하여 이식성을 높인다
